Project Route

프로젝트는 문제를 발견하고 기술로 연결한 경로입니다.

각 프로젝트를 결과물 목록이 아니라, 문제 인식에서 구현과 배운 점으로 이어지는 하나의 개발 기록으로 정리했습니다.

  1. Temi Project

    Temi 로봇 기반 서비스 프로젝트

    로봇이 사람과 만나는 상황을 상상하며 서비스 흐름을 설계한 프로젝트입니다. 단순히 기능을 붙이는 것이 아니라, 사용자가 로봇의 안내를 어떻게 받아들이고 다음 행동으로 이어지는지에 초점을 맞췄습니다.

    Role
    서비스 흐름 기획, 사용자 시나리오 정리, 기능 구조 설계
    Stack
    Temi, UI/UX, Service Planning
    Lesson
    기술이 실제 공간에 놓이면 사용자의 맥락이 더 중요해진다.

  2. Android App

    안드로이드 앱 개발

    모바일 환경에서 사용자가 빠르게 이해하고 조작할 수 있는 화면을 목표로 앱을 개발했습니다. 화면 전환, 입력 흐름, 상태 변화처럼 사용자가 직접 만지는 경험을 중심에 두고 기능을 구현했습니다.

    Stack
    Android, Kotlin, Java, Git
    Focus
    화면 구조, 사용자 입력, 기능 단위 구현, 반복 개선
    Lesson
    작은 인터랙션의 품질이 앱 전체의 신뢰감을 만든다.
